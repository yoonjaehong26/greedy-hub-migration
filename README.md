# greedy-hub

그리디(Greedy) 개발 동아리 공식 허브 — **설득·설명용 기획안(PRD)과 프로토타입**.

> ⚠️ 이 저장소는 동아리 메인테이너 설득·논의를 위한 **기획 단계 산출물**입니다. 실제 서비스 코드가 아니며, 프로토타입의 데이터는 데모용입니다.

## 📄 문서

- **[PRD (제품 요구사항 정의서)](docs/PRD.md)** — 문제정의·기능·아키텍처·로드맵·데이터모델. (버전은 PRD 상단/부록 E 참고)
- **[발표 제안서 (PPTX)](docs/greedy-hub-proposal.pptx)** — 메인테이너 설득용 슬라이드 (후킹 → 체계화 → 데모 → 요청).

## 🖥️ 프로토타입 (`prototype/`)

빌드 없이 도는 HTML+Tailwind 클릭 프로토타입. **화면 흐름·기능을 눈으로 보며 검증**하기 위한 설명용입니다.

- **배포 링크(라이브)**: **https://insane-p.github.io/greedy-hub/** ← 메인테이너 설득용, 바로 클릭해보기
  - ⚠️ 등장하는 이름·데이터는 전부 **데모용 가명/가상 데이터**입니다 (실제 동아리원 정보 아님).
- 로컬 실행: `prototype/` 폴더에서 `python -m http.server 4321` → http://localhost:4321 (또는 `index.html` 더블클릭)
- 우측 하단 **역할 전환기**로 외부인 · 멤버 · 리뷰어 · 운영진 · 이전 기수 시점을 바꿔볼 수 있고, 역할에 따라 탭이 달라집니다.

화면: 랜딩 · 미션(대시보드/상세) · 리뷰어 워크스페이스 · 멤버 디렉토리/이력서 · 기술블로그(목록/상세/작성) · 프로젝트(목록/상세) · 활동 타임라인 · 지원 · 운영진(대시보드/미션 출제/권한 관리).

## 🧭 핵심 방향 (요약)

- **3계층**: 웹 = 기록(진실의 원천) / GitHub = 산출물 / Discord(봇) = 알림·액션.
- **프론트**: Next.js(SEO) · **백엔드**: Spring Boot 3.x · JPA · springdoc(OpenAPI/Swagger) · **인증**: GitHub OAuth.
- 기존 `greedy-team/greedy-discord-bot`(리뷰어 매칭·구글폼 감시)은 재구현 없이 **통합**.
- 핵심 가치: 미션·리뷰 트래킹, 자동으로 쌓이는 **멤버 이력서**, 공개 **기술블로그** 아카이브.

> 다듬은 뒤 그리디 오거나이제이션으로 이관 예정.
